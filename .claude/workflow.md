# 下班上传工作流程

## 触发条件
当你说"下班了"时，自动执行以下流程：

## 上传步骤
1. **检查当前工作目录**
   - 查找所有修改过的文件
   - 识别新创建的文件
   - 检查未提交的更改

2. **整理commit信息**
   - 按功能分类提交
   - 自动生成有意义的commit信息
   - 格式："[功能类型] 具体描述"

3. **执行Git操作**
   - git add . (添加所有更改)
   - git commit (提交更改)
   - git push (推送到GitHub)

4. **生成工作报告**
   - 显示上传文件列表
   - 显示commit记录
   - 提供GitHub仓库链接

## Commit信息格式
- 页面开发：`[页面] XXX页面开发完成`
- 功能优化：`[优化] XXX功能优化`  
- Bug修复：`[修复] XXX问题修复`
- 资源更新：`[资源] XXX资源更新`
- 配置更改：`[配置] XXX配置调整`

## 使用示例

我会自动：
✅ 扫描所有更改
✅ 整理并上传代码
✅ 生成工作报告
